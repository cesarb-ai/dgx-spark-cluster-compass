# Security Policy

## Supported Versions

This repository is documentation-first and does not currently publish versioned
software releases with security patch guarantees. Security-relevant updates are
applied on a best-effort basis.

## Reporting a Vulnerability

If you discover a security issue related to scripts, examples, or documented
commands in this repository:

1. Do not open a public issue with exploit details.
2. Use GitHub Security Advisories (private report) for this repository.
3. Include:
   - A clear description of the issue
   - Affected file paths and command snippets
   - Reproduction steps
   - Potential impact and mitigation

We will acknowledge reports as soon as possible and work with you on
coordinated disclosure.
